**Methodology**:
- Detect CACM format and parse documents and queries.
- Build an in-memory inverted index (term postings and doc stats).
- Score queries using TF-IDF, Okapi TF, and BM25 (postings-based scoring for speed).
- Produce `cacm_staticFeatureMatrix.csv` and TREC-style run files in `cacm_outputs/`.
- Evaluate runs using an internal Python evaluator (produces `.eval.txt` files and an aggregated summary file).

**Project Structure** (key files):
- **Script**: [build_feature_matrix.py](build_feature_matrix.py) — orchestrates parsing, indexing, scoring, and file output.
- **Library**: [ir_pipeline.py](ir_pipeline.py) — parsing (CACM-aware), indexing, scoring, write helpers.
- **Legacy**: [Feature_Matrix.py](Feature_Matrix.py) — older AP-oriented code (patched for safety).

**Implementation Details**:
- **Parsing**: CACM parsing recognizes `.I` document boundaries and `.W` query text; query IDs normalized to match qrels.
- **Indexing**: `InMemoryIndex` stores term postings, doc lengths, collection frequency for efficient scoring.
- **Scoring**: Implemented TF-IDF (idf * tf), Okapi TF (document-length normalization), BM25 (k1=1.2,b=0.75). Laplace/JM are available but were excluded from the fast CACM run.
- **Evaluation**: A Python script computed MAP, nDCG@10, P@10 and wrote per-run `.eval.txt` files.
